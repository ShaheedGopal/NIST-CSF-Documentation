# NIST Cybersecurity Framework (CSF) Implementation Checklist

This checklist is based on the NIST Cybersecurity Framework (CSF) core functions: **Identify, Protect, Detect, Respond, and Recover**.  
It provides practical steps for aligning organizational cybersecurity practices with industry standards.

---

## 📌 Usage
This checklist can be used as:
- A self-assessment tool  
- A gap analysis reference  
- A planning guide for implementing security controls  

---

## ✅ Identify (ID)

| Subcategory | Description | Checklist |
|------------|------------|-----------|
| ID.AM-1 | Physical devices and systems inventoried | [ ] Maintain an up-to-date asset inventory |
| ID.AM-2 | Software platforms and applications inventoried | [ ] Track all software versions and licenses |
| ID.RA-1 | Asset vulnerabilities identified and documented | [ ] Conduct quarterly vulnerability scans |
| ID.RM-1 | Risk management process established | [ ] Document and approve a formal risk assessment policy |

**Additional Guidance:**
- Assign ownership for all assets  
- Maintain a centralized asset register  
- Regularly review and update risk assessments  

---

## 🛡️ Protect (PR)

| Subcategory | Description | Checklist |
|------------|------------|-----------|
| PR.AC-1 | Identities and credentials managed | [ ] Enforce MFA and password policies |
| PR.DS-1 | Data-at-rest is protected | [ ] Encrypt sensitive data on servers and endpoints |
| PR.IP-3 | Configuration change control in place | [ ] Maintain change control logs and approval workflows |
| PR.AT-1 | Security awareness training provided | [ ] Conduct annual cybersecurity awareness training |

**Additional Guidance:**
- Apply least privilege access controls  
- Implement endpoint protection solutions  
- Regularly patch systems and applications  

---

## 🔍 Detect (DE)

| Subcategory | Description | Checklist |
|------------|------------|-----------|
| DE.CM-1 | Network monitored to detect threats | [ ] Use IDS/IPS or firewall monitoring tools |
| DE.CM-7 | Monitoring for unauthorized devices | [ ] Perform periodic scans for rogue devices |
| DE.DP-4 | Detection events communicated | [ ] Configure alerts for suspicious activity |

**Additional Guidance:**
- Centralize logging (SIEM tools)  
- Define alert thresholds for suspicious activity  
- Regularly review logs and alerts  

---

## 🚨 Respond (RS)

| Subcategory | Description | Checklist |
|------------|------------|-----------|
| RS.RP-1 | Response plan executed during incident | [ ] Maintain and test an incident response plan |
| RS.CO-2 | Response coordinated with stakeholders | [ ] Define communication roles and escalation paths |
| RS.IM-1 | Lessons learned captured | [ ] Conduct post-incident reviews |

**Additional Guidance:**
- Document incident response procedures  
- Maintain contact lists for internal/external stakeholders  
- Perform simulations or tabletop exercises  

---

## 🔁 Recover (RC)

| Subcategory | Description | Checklist |
|------------|------------|-----------|
| RC.RP-1 | Recovery plan executed after incident | [ ] Document and test recovery plans quarterly |
| RC.IM-1 | Recovery incorporates lessons learned | [ ] Update recovery processes post-incident |
| RC.CO-1 | Communication strategy managed | [ ] Prepare internal and external communication templates |

**Additional Guidance:**
- Define Recovery Time Objectives (RTO)  
- Define Recovery Point Objectives (RPO)  
- Regularly test backup and restore procedures  

---

## 📊 Monitoring and Reporting

- Track implementation progress using dashboards (e.g., Power BI)  
- Report status to management and risk committees  
- Maintain evidence for audit and compliance  

---

## 📌 Compliance Alignment

This checklist supports alignment with:
- **NIST Cybersecurity Framework (CSF)**  
- **ISO/IEC 27001**  
- **POPIA / GDPR (where applicable)**
